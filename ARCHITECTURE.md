# CQI-MLAS 多模态课程持续改进系统 — 架构文档

## 1. 系统概述

**CQI-MLAS** (Course Quality Improvement — Multimodal Learning Analysis System) 是一款面向高校课程持续改进机制的多模态数据分析平台。

系统以**课程**为中心，通过接入学习通、智慧树、线下监控摄像头、自有教学平台等多源数据，利用多模态融合技术对课程运行全过程进行分析，实现基于 OBE 理念和 PDCA 循环的课程持续改进。

### 核心理念
- **课程持续改进 (CQI)**：不是给学生画像，而是给课程画像
- **多模态数据融合**：视频微表情 + 文本语义 + 交互行为 + 传统学习数据
- **六维课程画像**：学生学的状态 · 老师教的状态 · 平台资源质量 · 教学互动方式 · 教学方法适配 · 教学环境
- **人机协同决策**：AI 分析 + 教师专业判断
- **PDCA 闭环**：计划 → 执行 → 检查 → 处理

## 4. 六维课程画像详解

| 维度 | 核心指标 | 数据来源 | 分析方法 |
|------|---------|---------|---------|
| **学生学的状态** | 视频专注度、弹幕互动率、情绪分布、综合投入度 | 摄像头、学习通 | 视频微表情分析、文本情感分析 |
| **老师教的状态** | 讲授语速、情绪饱满度、走动频次、眼神交流、提问频次 | 摄像头、自有平台 | 行为模式识别、语音分析 |
| **平台资源质量** | 课件完播率、难点回放率、视频观看深度、资源满意度 | 学习通、智慧树 | 使用行为分析、热力图 |
| **教学互动方式** | 小组讨论热度、师生问答、翻转课堂参与率、实时投票 | 学习通、智慧树 | 社交网络分析、互动深度评估 |
| **教学方法适配** | 案例教学、翻转课堂、项目驱动、支架式教学等8种方法 | 多源数据融合 | 教学方法有效性评估 |
| **教学环境** | 温度舒适度、光照、声学质量、设备完好率、容量利用率 | 传感器、问卷 | 环境舒适度评估 |

## 5. 前端路由

| 路由 | 页面 | 功能 |
|------|------|------|
| `/` | 课程全局画像 | 六维指标概览、健康度趋势、实时数据流 |
| `/collection` | 多源数据采集 | 四维数据采集配置与监控 |
| `/datahub` | 数据源管理 | 学习通/智慧树/摄像头等数据源统一管理 |
| `/fusion` | 多模态数据融合 | 早融合/晚融合策略对比、权重配置 |
| `/diagnosis` | 动态诊断 | O2O混合式教学断点诊断 |
| `/ai` | AI决策支持 | 人机协同课程优化决策 |
| `/vitality` | 课程生命力 | 五维生命力评估与可视化 |
| `/improvement` | 持续改进(CQI+PDCA) | CQI闭环流程与改进证据 |
| `/profile` | 课程画像详情 | 六维画像详情、教学方法适配度 |
| `/efficacy` | 干预成效验证 | Pre/Post 对比、研究结论 |
